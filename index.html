1. Introduction

This page explains how TLRI Ltd ("we", "us", "our") uses the Allegro REST API across our internal Allegro applications. Both applications are used exclusively for internal business operations at Vilbury London and are not distributed to external users.

2. Application Purpose

We operate two registered Allegro applications, both used as internal business tools:

2.1 Vilbury scripts (main integration)

Used to:

  • Create, update, and manage offers (listings) on Allegro marketplaces
  • Synchronize stock levels and pricing between our inventory system and Allegro
  • Process and fulfill customer orders received via Allegro
  • Reconcile billing and payment data for accounting and VAT compliance
  • Maintain accurate product information and seller account settings

2.2 Temporal search products (catalog search)

A separate application used exclusively to:

  • Search the Allegro product catalog by GTIN/EAN to match our SKUs to Allegro product IDs
  • Retrieve product details and required parameters before creating new offers

It is registered as a separate app so that catalog-search traffic uses its own rate-limit bucket and cannot interfere with the main integration's quota.

Important: Neither application is available to external users or third parties. Both are used solely by authorized TLRI Ltd personnel for business operations on TLRI Ltd's own Allegro seller accounts.

3. Data Collection

3.1 What Data We Access

Through Allegro's REST API, our application accesses:

  • Offer Data: Product listings, SKUs, titles, descriptions, pricing, quantity, images, parameters, categories
  • Order Data: Order details, buyer shipping information, order status, fulfillment data
  • Billing Data: Billing entries, invoices, fees and commissions
  • Payment Data: Payment operations, settlements, transaction records
  • Account Data: Seller profile information from the /me endpoint, sale settings

3.2 Authentication Data

We store:

  • OAuth 2.0 refresh tokens (encrypted, used to authenticate with the Allegro API)
  • Access tokens (temporary, auto-refreshed before expiry)

4. How We Use Data

Data accessed through the Allegro API is used to:

  • Offer Management: Create and update product listings, synchronize stock levels and pricing
  • Order Fulfillment: Process customer orders, generate shipping documents, track deliveries
  • Financial Reconciliation: Match Allegro billing entries and payment operations against our accounting system
  • Business Analytics: Generate internal reports on sales performance and inventory turnover
  • Compliance: Ensure accurate VAT settings and regulatory compliance across Allegro marketplaces

5. Data Storage and Security

5.1 Where Data Is Stored

  • Primary Database: Supabase (PostgreSQL) - cloud-hosted with encryption at rest
  • Server Location: Data centers in the European Union (EU)
  • Local Cache: Temporary token cache files on secure company servers

5.2 Security Measures

  • All API communications use HTTPS/TLS encryption
  • OAuth 2.0 authentication with rotating access tokens
  • Database access restricted to authorized personnel only
  • Regular security updates and monitoring
  • Access tokens auto-refreshed before expiry
  • Refresh tokens rotated regularly per Allegro's token lifecycle

6. Data Sharing

We do not share, sell, or rent any Allegro data to third parties.

Data access is strictly limited to:

  • Authorized TLRI Ltd employees who require access for business operations
  • Supabase (our database provider) - covered under their data processing agreement

7. Data Retention

  • Offer Data: Retained while listings are active, plus 12 months for historical records
  • Order Data: Retained for 7 years to comply with UK and Polish tax and accounting regulations
  • Billing & Payment Data: Retained for 7 years for accounting and audit purposes
  • Authentication Tokens: Access tokens deleted after expiry; refresh tokens retained until replaced

13. Allegro Developer Program Compliance

This application is provided in compliance with Allegro Developer Program requirements. Our application:

  • Uses OAuth 2.0 (Authorization Code flow) for secure authentication
  • Requests only the minimum necessary permissions (scopes)
  • Implements industry-standard security practices
  • Complies with Allegro API rate limits (9000 requests per minute) and usage policies

Application Scopes Requested:

  • allegro:api:profile:read (read seller profile / /me endpoint)
  • allegro:api:sale:offers:read (read offers and listings)
  • allegro:api:sale:offers:write (create and manage offers)
  • allegro:api:sale:settings:read (read sale settings)
  • allegro:api:sale:settings:write (manage sale settings)
  • allegro:api:orders:read (read orders for fulfillment)
  • allegro:api:billing (read billing entries and fees)
  • allegro:api:payments (read payment operations and settlements)

This document applies exclusively to TLRI Ltd's internal use of the Allegro API for business operations.
